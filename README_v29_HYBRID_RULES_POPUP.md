# v29 Hybrid Rules + Summary Popup

## 주요 변경

1. 분류 엔진을 점수 우선 방식에서 **hard rule 우선 + 보조 점수** 방식으로 수정했습니다.
   - 회수/처분: 회수, 행정처분, 판매중지, 영업정지, 허가취소 등 명확한 사건형 단어가 있어야 대표분류/레이더에 표시됩니다.
   - 정책/가이드라인: 규제기관명 + 지침/가이드라인/고시/행정예고/개정 등 정책성 단어 조합으로 판단합니다.
   - 허가/임상: FDA IND 승인류는 정책/가이드라인으로 오분류되지 않도록 대표분류를 허가/임상으로 우선 처리합니다.

2. 기사 요약 표시를 카드 하단 펼침형에서 **팝업형 보드 UI**로 변경했습니다.
   - 원문 열기 옆의 `기사 요약 보기` 버튼 클릭 시 팝업이 열립니다.
   - 업로드하신 보드 이미지 스타일을 배경으로 사용하고, 보드 영역에 요약/태그/분류근거가 표시됩니다.

3. 가짜 요약을 제거했습니다.
   - 원문 본문 수집 실패 + RSS 요약 부족 시 제목 반복을 하지 않습니다.
   - 대신 요약 생성 불가 및 본문수집상태를 명확히 표시합니다.

4. 규제 레이더 노출을 개선했습니다.
   - 대표분류뿐 아니라 다중태그 기준으로도 레이더 lane에 포함됩니다.
   - 같은 URL/유사 제목 반복 노출을 줄이고, lane별 이슈 다양성을 우선합니다.

## 사용 시 확인사항

- 기존 Supabase 컬럼은 v28 migration과 동일하게 사용합니다. 추가 migration은 필요 없습니다.
- 예전 캐시가 남아 있으면 앱 첫 실행 시 force reclassify가 적용되지만, Supabase에 저장된 오래된 article_summary는 기존 값이 남아 있을 수 있습니다. 필요 시 새로 수집하거나 로컬 CSV를 삭제 후 재수집하십시오.
