# v33 KST 미래시간 과보정 방어

## 반영 배경
v32에서 날짜 선택창은 KST 기준으로 보정했으나, 일부 Google News RSS/Supabase 캐시 데이터가 실제 KST 시각을 UTC 시각처럼 저장하거나 해석하면서 화면에 현재시각보다 9시간 앞선 시간이 표시되는 문제가 있었습니다.

예: KST 현재가 2026-05-18 08:57인데 기사 발행시각이 2026-05-18 17:24로 표시되는 현상

## 수정 내용
1. `modules/time_utils.py`에 미래시간 방어 로직 추가
   - KST 변환 후 현재 KST보다 30분 이상 미래인 뉴스 발행시각은 UTC/KST 오인식 가능성이 높은 값으로 판단
   - 이 경우 9시간을 차감하여 실제 KST 발행시각으로 보정

2. 기존 Supabase 캐시 보정
   - Supabase에 이미 `08:24+00:00` 형태로 저장된 기존 데이터가 있어도 화면 표시 시 `17:24`가 아니라 `08:24`로 보정

3. 신규 RSS 수집 데이터 보정
   - Google News RSS `published_parsed`, `published` 값에서 같은 문제가 발생해도 최종 표시/저장 전 `to_kst_timestamp()` 경유 시 보정

## 검증 예시
- `2026-05-18T08:24:20+00:00` → `2026-05-18 08:24:20 KST`
- `2026-05-18 17:24:20` → 현재보다 미래이면 `2026-05-18 08:24:20 KST`
- `2026-05-17T23:24:20+00:00` → `2026-05-18 08:24:20 KST`

## 주의
이 보정은 뉴스 발행시각에 대해서만 현실적으로 적용되는 방어 로직입니다. 뉴스가 현재보다 30분 이상 미래로 발행되는 것은 정상적인 케이스가 아니므로, 9시간 과보정으로 판단합니다.
